  STEP-BY-STEP IMPLEMENTATION

  TASK 1: Set Up PostgreSQL Database

  Option A: Using Supabase (Recommended for Demo)

  1. Go to https://supabase.com and sign up/login
  2. Create a new project:
    - Project name: aleev-demo or similar
    - Database password: Generate a strong password (save it!)
    - Region: Choose closest to you
    - Wait 2-3 minutes for project provisioning
  3. Get connection strings:
    - Click "Project Settings" (gear icon) → "Database"
    - Under "Connection string" section, you'll see:
        - URI: Copy the "URI" connection string (starts with
  postgresql://postgres:[YOUR-PASSWORD]@)
      - Pooler (Transaction Mode): Copy this for DATABASE_URL
      - Direct Connection: Use the URI for DATABASE_DIRECT_URL
  4. Important: Replace [YOUR-PASSWORD] in both URLs with your actual
  password

  Example URLs:
  DATABASE_URL="postgresql://postgres.abcdefghijklmnop:your-password@aws-0-u
  s-west-1.pooler.supabase.com:6543/postgres?pgbouncer=true"
  DATABASE_DIRECT_URL="postgresql://postgres:your-password@db.abcdefghijklmn
  op.supabase.co:5432/postgres"

  Option B: Using Neon

  1. Go to https://neon.tech and sign up/login
  2. Create new project: "aleev-demo"
  3. Copy the connection string from dashboard
  4. Use the SAME connection string for both DATABASE_URL and
  DATABASE_DIRECT_URL

  Option C: Local PostgreSQL

  # Install PostgreSQL (macOS)
  brew install postgresql@14
  brew services start postgresql@14

  # Create database
  createdb aireview

  # Connection strings
  DATABASE_URL="postgresql://localhost:5432/aireview"
  DATABASE_DIRECT_URL="postgresql://localhost:5432/aireview"

  TROUBLESHOOTING

  Issue: "Error: P1001: Can't reach database server"

  Cause: Database connection string is incorrect or database is not
  accessible

  Solutions:
  1. Verify DATABASE_DIRECT_URL in .env.local
  2. Check database is running (for local PostgreSQL)
  3. Test connection manually:
  psql "postgresql://your-connection-string-here"
  4. For Supabase: Check IP restrictions in dashboard
  5. Check for firewall blocking port 5432

  Issue: "Error: Environment variable not found: DATABASE_DIRECT_URL"

  Cause: .env.local is missing DATABASE_DIRECT_URL

  Solution: Add DATABASE_DIRECT_URL to .env.local (same as DATABASE_URL for
  most providers)

  Issue: "Error: Authentication failed for user"

  Cause: Password in connection string is incorrect

  Solutions:
  1. Verify password in Supabase/Neon dashboard
  2. Check for special characters needing URL encoding:
    - Replace @ with %40
    - Replace # with %23
    - Replace $ with %24

  Issue: Seed script runs but no data appears

  Cause: Seed script might be using wrong database

  Solution:
  # Clear database and re-seed
  npx prisma migrate reset --force
  npm run db:seed

  Issue: "Module not found: Can't resolve '@prisma/client'"

  Cause: Prisma Client not generated

  Solution:
  npm run db:generate
