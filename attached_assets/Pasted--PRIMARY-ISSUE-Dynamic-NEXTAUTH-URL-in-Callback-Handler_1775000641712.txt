🔴 PRIMARY ISSUE: Dynamic NEXTAUTH_URL in Callback Handler

  Location: src/app/api/auth/[...nextauth]/route.ts:8-13

  async function authHandler(req: NextRequest, context: any) {
    const host = req.headers.get("x-forwarded-host") ||
  req.headers.get("host");
    const proto = req.headers.get("x-forwarded-proto") || "https";

    if (host && proto) {
      process.env.NEXTAUTH_URL = `${proto}://${host}`;  // ⚠️ PROBLEMATIC
    }
    return handler(req, context);
  }

  Why this causes loops:
  - NextAuth calculates callback URLs and cookie domains based on
  NEXTAUTH_URL
  - Dynamically changing this value between requests creates inconsistencies
  - If headers differ between the initial OAuth request and the callback
  request, NextAuth can't match them
  - The session cookie might be set for the wrong domain or path
  - User ends up back at login page without a valid session → loop

  🔧 Recommended Fixes (in order of priority)

  Fix 1: Remove Dynamic NEXTAUTH_URL Setting (CRITICAL)

  The NEXTAUTH_URL should be static and set via environment variables only.
  Remove the dynamic setting from the route handler:

  In src/app/api/auth/[...nextauth]/route.ts, change to:
  import { NextRequest } from "next/server";
  import NextAuth from "next-auth";
  import { authOptions } from "@/lib/auth";

  const handler = NextAuth(authOptions);

  export { handler as GET, handler as POST };

  Make sure your production environment has NEXTAUTH_URL set correctly in
  environment variables (not dynamically).
